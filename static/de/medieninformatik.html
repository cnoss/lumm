    Themenvorschlag: Atomic Design Implementierung

    Die Methodologie Atomic Design ist ein eleganter Ansatz mit einem sehr prägnantem Leitbild für GUI-Modularisierung im Web. Doch wie lässt sich dieser Ansatz in verschiedenen Frameworks und Content Management Systemen umsetzen?

    Eine Abschlussarbeit in diesem Themenfeld sollte die Methodologie beschreiben, gegen andere Ansätze abgrenzen und deren Vor- und Nachteile beleuchten. In einem zweiten Schritt könnte dann ein Beispielprojekt in einem CMS oder Framework durchgeführt werden, wobei der Fokus auf der Implementierung der Methodologie läge.

    Mögliche CMSe? Im Grunde offen. Schön wäre mal eine Implementierung in Wordpress.
    Mögliche Frameworks? Auch offen. Angular, Ractive.js und React wären interessant.

    Mockup 2 Markup

    Auf der Suche nach einem geschmeidigen Übergang von Design(mockups) zu interaktiven Prototypen.

    Zur Zeit fokussiere ich mich bei meiner Hochschularbeit auf das Thema "Übergang von Design(Mockups) zu interaktiven Prototypen" im Kontext Web-basierter Anwendungen. Es gibt verschiedene Vorgehensweisen um von einer Idee zum fertigen Web-Projekt zu kommen und egal, ob man Ansätzen wie "Mobile First" folgt, ob man entlang von Workflows wie dem "Responsive Design Workflow" arbeitet oder Methodologien wie SMACSS, OOCSS, Atomic Web Design oder BEM nutzt: irgendwann muss eine irgendwie visualisierte Idee in Code umgesetzt werden. Um diesen Übergang geht es beim Projekt Mockup2Markup.

    Zu dieser Fragestellung hab ich ein Projekt auf Github platziert. Ich bin noch auf der Suche nach Mitstreitern und Supporten. Mitmachen macht Spaß. Und schlau :)
